## Task 2 · StateDB 权重访问 API

**目的**：给 actuator 一个能原子更新 `total_net_weight` / `total_energy_weight` 的入口。

**动作**
- `core/state/dynamic_properties.go`：若 M1.1 只加了 getter 没加 `Add*` 风格的增量 setter，在这补 `AddTotalNetWeight(delta int64)` / `AddTotalEnergyWeight(delta int64)`（用现有 setter + getter 组合实现，内部做一次 overflow sanity check）。

**验收**：`core/state/dynamic_properties_test.go` 新增 `TestAddTotalNetWeight_RoundTrip`。

## Task 5 · Freeze/Unfreeze Actuator 权重同步 + fork gate

**动作**
- `actuator/freeze_balance.go`：
  - Validate 入口加 `allow_new_resource_model` gate：激活后 `return ErrV1FreezeClosed`（错误字符串对齐 java 文案）。
  - Execute 完成 StateDB 写入后，按 `allow_new_reward` 分支调 `AddTotalNetWeight(+delta)` / `AddTotalEnergyWeight(+delta)`。
- `actuator/unfreeze_balance.go`：
  - 对过期条目 iteration 后统一调 `Add*(-removedAmount/TRX_PRECISION)`；不加 fork gate（允许 post-fork unfreeze）。
- 错误路径：权重更新失败（overflow）时 Execute 返回 error，与 java-tron 一致（java 其实直接 throw，等价）。

**验收**
- `actuator/freeze_v1_weight_test.go`：freeze/unfreeze 前后 DP 权重变化。
- `actuator/unfreeze_v1_postfork_test.go`：fork=1 仍可 unfreeze。
- 已有 `actuator/freeze_balance_test.go` 分两组 fork 状态跑。
